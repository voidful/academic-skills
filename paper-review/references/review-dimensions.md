# 五大審查維度：完整說明

本文件詳細說明學術論文審稿的五大核心維度，每個維度包含具體檢查項目以及好論文與差論文的特徵對比。

---

## 維度 1：貢獻新穎性 (Novelty & Contribution)

### 定義

評估論文所提出的想法、方法或發現是否具有足夠的新穎性與貢獻度，能為該領域帶來有意義的推進。

### 具體檢查項目

1. **原創性**
   - 核心想法是否為首次提出？
   - 如果是現有方法的延伸，延伸的幅度是否足夠？
   - 是否只是把 A 領域的已知方法搬到 B 領域？

2. **貢獻的類型與層次**
   - 技術貢獻：新演算法、新架構、新理論結果
   - 應用貢獻：將已有方法應用於重要的新問題
   - 洞見貢獻：揭示了之前未知的現象或理解
   - 資源貢獻：新資料集、新 benchmark、新工具

3. **相對於 Prior Work 的定位**
   - 是否清楚說明與最相近工作的區別？
   - 增量改進是否足以構成獨立貢獻？
   - 相關工作的討論是否全面？

4. **影響力預估**
   - 這個貢獻是否會影響後續研究？
   - 其他研究者是否會採用或引用這個方法？
   - 是否開啟了新的研究方向？

### 好論文 vs 差論文

| 面向 | 好論文特徵 | 差論文特徵 |
|------|-----------|-----------|
| 想法 | 令人驚訝或反直覺的洞見 | 顯而易見或 trivial 的想法 |
| 定位 | 清楚說明 gap 在哪裡 | 找不到與 prior work 的區別 |
| 增量 | 質的突破或顯著的量的進步 | 微小改進或換湯不換藥 |
| 影響 | 可能改變領域方向 | 不太會有後續影響 |

## 維度 2：寫作清晰度 (Clarity & Writing)

### 定義

評估論文的表達是否清晰、邏輯是否連貫、讀者是否能順暢地理解論文的主要貢獻與技術細節。

### 具體檢查項目

1. **結構與邏輯流**
   - 論文結構是否標準且合理？
   - 段落之間的過渡是否自然？
   - 讀者是否能沿著作者的思路前進？

2. **動機與故事線**
   - 為什麼這個問題重要？是否有說服力地說明了？
   - 從問題到方法的推導是否自然？
   - 讀者在讀完導論後是否清楚論文要做什麼？

3. **技術表達**
   - 符號系統是否一致？
   - 數學公式是否有充分的文字解釋？
   - 術語是否有明確定義？

4. **圖表品質**
   - 圖表是否清晰、美觀、資訊豐富？
   - 圖表的 caption 是否自足（self-contained）？
   - 圖表是否有在正文中被引用和討論？

5. **語言品質**
   - 文法是否正確？
   - 用詞是否精確？
   - 是否避免了不必要的冗餘？

### 好論文 vs 差論文

| 面向 | 好論文特徵 | 差論文特徵 |
|------|-----------|-----------|
| 動機 | 開頭就抓住讀者，清楚說明 why | 動機不明或過於空泛 |
| 邏輯 | 每段都自然承接上段 | 邏輯跳躍，需要猜測作者意圖 |
| 符號 | 一致、簡潔、有直覺 | 混亂、前後不一、過度複雜 |
| 圖表 | 一圖勝千言 | 圖表模糊或多餘 |

## 維度 3：實驗嚴謹性 (Experimental Rigor)

### 定義

評估實驗設計是否嚴謹、結果是否可靠、能否充分支持論文的 claims。

### 具體檢查項目

1. **實驗設計**
   - 實驗是否針對論文的核心 claims 設計？
   - 是否有控制變量（controlled experiments）？
   - 實驗設定（setup）是否描述得足夠詳細以供重現？

2. **Baselines 的選擇**
   - 是否包含了最新且最強的 baselines？
   - 是否包含了最經典的 baselines？
   - Baselines 是否經過合理的超參數調整？
   - 是否避免了 straw man baselines？

3. **統計可靠性**
   - 是否報告了多次實驗的平均值與標準差？
   - 是否進行了統計顯著性檢驗？
   - 樣本量是否足夠？

4. **公平性**
   - 所有方法是否使用相同的資料劃分？
   - 計算資源是否可比？
   - 超參數選擇是否對所有方法公平？

5. **可重現性**
   - 是否提供了程式碼或虛擬碼？
   - 超參數是否完整列出？
   - 隨機種子是否固定？

### 好論文 vs 差論文

| 面向 | 好論文特徵 | 差論文特徵 |
|------|-----------|-----------|
| 設計 | 每個實驗都針對一個 claim | 實驗與 claims 脫節 |
| Baselines | 包含 SOTA 且公平比較 | 缺少關鍵 baselines 或不公平比較 |
| 統計 | 有誤差線和顯著性檢驗 | 只報告單次結果 |
| 可重現 | 細節充分，附有程式碼 | 缺少關鍵實驗細節 |

## 維度 4：評估完整性 (Evaluation Completeness)

### 定義

評估論文的實驗評估是否全面，是否涵蓋了驗證其 claims 所需的各個面向。

### 具體檢查項目

1. **資料集覆蓋**
   - 是否在多個資料集上進行了實驗？
   - 資料集是否涵蓋了不同的場景和難度？
   - 是否包含了該領域公認的標準 benchmark？

2. **Ablation Study**
   - 是否對每個關鍵設計選擇進行了 ablation？
   - 各個組件的貢獻是否被獨立驗證？
   - Ablation 的設計是否合理（一次只改一個變量）？

3. **分析的深度**
   - 是否有 error analysis？
   - 是否有 case study 或 qualitative analysis？
   - 是否分析了方法在不同條件下的表現？
   - 是否討論了 failure cases？

4. **效率分析**
   - 是否報告了計算成本（時間、記憶體、FLOPs）？
   - 是否與 baselines 進行了效率比較？
   - 是否分析了可擴展性（scalability）？

5. **局限性討論**
   - 是否誠實地討論了方法的局限性？
   - 是否指出了方法不適用的場景？
   - 是否討論了潛在的負面社會影響？

### 好論文 vs 差論文

| 面向 | 好論文特徵 | 差論文特徵 |
|------|-----------|-----------|
| 資料集 | 多樣且標準的 benchmarks | 只在一兩個資料集上測試 |
| Ablation | 全面且有洞見 | 缺少 ablation 或設計不當 |
| 分析 | 深入的 error analysis 和 case study | 只有數字沒有分析 |
| 局限 | 誠實討論局限性 | 迴避或完全不提局限性 |

## 維度 5：方法健全性 (Soundness of Method)

### 定義

評估論文所提出的方法在技術層面是否正確、假設是否合理、理論基礎是否穩固。

### 具體檢查項目

1. **數學正確性**
   - 定理的證明是否正確？
   - 推導過程是否有跳步或錯誤？
   - 符號使用是否一致？

2. **假設的合理性**
   - 所有假設是否被明確陳述？
   - 假設在實際應用中是否成立？
   - 違反假設時方法是否仍然有效？

3. **方法的完整性**
   - 方法描述是否完整到可以重新實作？
   - 是否有未說明的設計選擇？
   - 邊界情況（edge cases）是否被處理？

4. **理論與實踐的一致性**
   - 理論分析是否支持實驗結果？
   - 理論保證是否在實際條件下有意義？
   - 是否有理論與實驗不一致的地方？

5. **潛在的技術問題**
   - 是否有 data leakage？
   - 是否有 overfitting 的風險？
   - 優化問題是否保證收斂？
   - 是否有數值穩定性的問題？

### 好論文 vs 差論文

| 面向 | 好論文特徵 | 差論文特徵 |
|------|-----------|-----------|
| 正確性 | 推導嚴謹無誤 | 有數學錯誤或邏輯漏洞 |
| 假設 | 明確且合理 | 隱含不合理的假設 |
| 完整性 | 可以據此重新實作 | 缺少關鍵細節 |
| 一致性 | 理論與實驗互相支持 | 理論與實驗脫節 |

---

## 維度間的交互關係

各維度不是獨立的——它們之間有重要的交互關係：

- **新穎性 x 健全性**：非常新穎但不健全的方法是危險的（誤導後續研究）
- **清晰度 x 其他所有維度**：寫作不清楚會讓所有其他維度都難以評估
- **嚴謹性 x 評估完整性**：嚴謹但不完整的實驗可能遺漏重要發現
- **健全性 x 實驗**：理論上健全但實驗不支持，可能意味著理論有隱含問題

### 致命缺陷的辨識

某些維度的嚴重問題可以直接構成拒稿理由：

1. **方法有根本性錯誤**（健全性 = 1）：無論其他維度多好都應拒稿
2. **完全沒有新意**（新穎性 = 1）：除非實驗極其全面且有重大實用價值
3. **實驗完全不可信**（嚴謹性 = 1）：結論失去支撐，論文站不住腳
