# Ablation Study 設計指南

## 概述

Ablation Study（消融實驗）是學術論文中不可或缺的實驗類型，用以驗證所提出方法中各組件的貢獻，並深入理解模型的行為特性。本文件詳細說明四種 Ablation 設計模式。

---

## 模式一：完整消融（Component Ablation）

### 核心概念

逐一移除或替換方法中的各個組件，觀察每個組件對整體效能的影響。這是最基本也是最重要的 Ablation 模式。

### 設計原則

1. **逐一移除**：每次實驗僅移除或替換一個組件
2. **合理替代**：移除組件時，需用合理的替代方案填補（而非直接刪除導致模型結構不完整）
3. **完整覆蓋**：方法中所有新提出的組件都應被消融

### 設計步驟

**第一步：列出所有組件**

將你的方法分解為獨立的組件列表。例如：

```
完整方法 = 組件 A + 組件 B + 組件 C + 組件 D
```

**第二步：設計消融變體**

| 變體名稱 | 組件 A | 組件 B | 組件 C | 組件 D |
|----------|--------|--------|--------|--------|
| 完整模型 | V | V | V | V |
| w/o A | X | V | V | V |
| w/o B | V | X | V | V |
| w/o C | V | V | X | V |
| w/o D | V | V | V | X |
| Baseline | X | X | X | X |

**第三步：定義「移除」的具體操作**

每個組件被移除時，需明確描述：

- 用什麼替代？（例如：用標準注意力替代提出的跨注意力）
- 模型結構如何調整？
- 參數量是否改變？

**第四步：分析結果**

- 計算每個組件的貢獻度（移除後的效能下降量）
- 排列組件重要性順序
- 檢查是否存在組件間的交互效應

### 進階：組合消融

若組件之間可能存在交互效應，可增加組合消融：

| 變體名稱 | 組件 A | 組件 B | 組件 C |
|----------|--------|--------|--------|
| 完整模型 | V | V | V |
| w/o A | X | V | V |
| w/o B | V | X | V |
| w/o C | V | V | X |
| w/o A+B | X | X | V |
| w/o A+C | X | V | X |
| w/o B+C | V | X | X |
| Baseline | X | X | X |

注意：組合消融的實驗數量會隨組件數指數增長，需權衡實驗成本。

### 結果呈現建議

- 使用表格呈現各變體的效能
- 使用柱狀圖視覺化各組件的貢獻度
- 討論各組件的相對重要性

## 模式二：超參敏感度分析（Hyperparameter Sensitivity）

### 核心概念

探討關鍵超參數對方法效能的影響，幫助讀者理解方法的穩健性與超參數選擇的敏感程度。

### 設計原則

1. **聚焦關鍵超參數**：選擇最可能影響效能的 2-4 個超參數
2. **合理取值範圍**：基於理論或經驗設定搜索範圍
3. **控制其他變數**：分析每個超參數時，固定其他超參數為最佳值

### 超參數選擇策略

**優先分析的超參數類型**：

| 優先級 | 超參數類型 | 範例 |
|--------|-----------|------|
| 高 | 方法特有的新超參數 | 提出的損失函數中的權重係數 |
| 高 | 對效能影響大的超參數 | 學習率、隱藏層維度 |
| 中 | 結構相關的超參數 | 層數、注意力頭數 |
| 低 | 通用的訓練超參數 | 批次大小、dropout 率 |

### 設計步驟

**第一步：選擇目標超參數**

從方法中識別 2-4 個最關鍵的超參數。

**第二步：設定取值範圍**

每個超參數設定 5-7 個取值點，建議包含：

- 極小值（可能不足的值）
- 較小值
- 中等值（通常接近最佳值）
- 較大值
- 極大值（可能過大的值）

**第三步：逐一變化實驗**

固定其他超參數，僅變化目標超參數：

```
超參數 λ 的敏感度分析：
λ = 0.01  → F1 = 85.2
λ = 0.05  → F1 = 87.1
λ = 0.1   → F1 = 88.5  ← 最佳值
λ = 0.5   → F1 = 87.8
λ = 1.0   → F1 = 84.3
```

**第四步：繪製敏感度曲線**

- X 軸：超參數值
- Y 軸：評估指標
- 標註最佳值與合理範圍

### 結果分析要點

- **穩健性**：效能是否對超參數值敏感？（平坦曲線 = 穩健）
- **最佳區間**：表現良好的超參數範圍有多寬？
- **極端行為**：在極端取值下模型的行為如何？
- **實用建議**：為讀者提供超參數選擇的建議

### 結果呈現建議

- 使用折線圖呈現每個超參數的敏感度曲線
- 多個超參數可使用子圖並排展示
- 在圖中標註最佳值與建議範圍
- 若有兩個超參數間的交互作用，可使用熱力圖

## 模式三：跨資料集遷移（Cross-Dataset Transfer）

### 核心概念

在多個不同的資料集上測試方法，驗證其泛化能力與適用範圍。

### 設計原則

1. **多樣性**：選擇具有不同特性的資料集
2. **代表性**：資料集應覆蓋該任務的主要變體
3. **公認性**：優先選擇該領域廣泛使用的基準資料集

### 資料集選擇策略

**差異維度**：

| 維度 | 說明 | 範例 |
|------|------|------|
| 規模 | 不同大小的資料集 | 小（1K）、中（10K）、大（100K+） |
| 領域 | 不同主題領域 | 新聞、生醫、法律、社群媒體 |
| 語言 | 不同語言或語種 | 中文、英文、多語言 |
| 難度 | 不同難度等級 | 簡單分類、細粒度分類 |
| 分佈 | 不同的標籤分佈 | 均衡、不均衡 |

### 設計步驟

**第一步：選擇資料集組合**

選擇 3-5 個資料集，確保覆蓋多個差異維度：

```
主資料集：SQuAD 2.0（英文、大規模、標準難度）
遷移資料集 1：TriviaQA（英文、大規模、不同來源）
遷移資料集 2：CMRC（中文、中規模）
遷移資料集 3：BioASQ（英文、小規模、生醫領域）
```

**第二步：設定實驗條件**

- 在主資料集上訓練，在遷移資料集上測試（Zero-Shot Transfer）
- 在每個資料集上分別訓練與測試（In-Domain Performance）
- 在主資料集上訓練，在遷移資料集上微調（Few-Shot Transfer）

**第三步：分析泛化模式**

- 方法在哪些資料集上表現最佳？為什麼？
- 方法在哪些資料集上表現不佳？為什麼？
- 泛化能力與資料集特性的關係是什麼？

### 結果分析要點

- **一致性**：方法是否在所有資料集上都優於 Baseline？
- **差異性**：不同資料集上的效能差異有多大？
- **條件**：方法在什麼條件下最有效？什麼條件下失效？
- **實用性**：方法是否可以直接應用於新領域？

### 結果呈現建議

- 使用表格呈現各資料集上的效能
- 計算跨資料集的平均表現與標準差
- 使用雷達圖展示各資料集的相對效能
- 深入分析效能差異的原因

## 模式四：定性分析（Qualitative Analysis）

### 核心概念

透過可視化與案例分析，深入理解模型的行為特性，提供量化指標無法傳達的洞見。

### 設計原則

1. **互補性**：定性分析應與定量結果互補，提供更深層的理解
2. **代表性**：選擇的案例應具有代表性，而非只展示最好的結果
3. **系統性**：按照一定的框架進行分析，而非隨意選取

### 分析類型

#### 4.1 注意力權重視覺化

**適用場景**：使用注意力機制的模型

**設計方法**：

1. 選擇具有代表性的輸入樣本
2. 提取模型各層的注意力權重
3. 使用熱力圖展示注意力分佈
4. 比較不同模型變體的注意力模式

**分析要點**：

- 模型是否關注了語義上重要的部分？
- 不同層的注意力模式有何差異？
- 你提出的機制是否改變了注意力的分佈方式？

#### 4.2 成功與失敗案例分析

**適用場景**：所有類型的模型

**設計方法**：

1. 從測試集中選取案例，按以下類別分類：
   - 我們的方法正確，Baseline 錯誤（展示優勢）
   - 我們的方法錯誤，Baseline 正確（展示劣勢）
   - 兩者都正確（簡單案例）
   - 兩者都錯誤（困難案例）
2. 每個類別選取 2-3 個代表性案例
3. 深入分析每個案例的模型行為

**分析要點**：

- 你的方法在什麼類型的輸入上表現更好？
- 失敗案例有什麼共同特徵？
- 失敗案例是否暗示了未來改進的方向？

#### 4.3 特徵空間視覺化

**適用場景**：需要理解模型學到的表示

**設計方法**：

1. 從模型中提取中間層的特徵表示
2. 使用降維方法（t-SNE、UMAP、PCA）將高維特徵映射到 2D
3. 用不同顏色標記不同類別或屬性
4. 比較不同模型變體的特徵空間

**分析要點**：

- 不同類別的特徵是否能良好分離？
- 你的方法是否產生了更有區分力的特徵？
- 特徵空間中是否有明顯的聚類結構？

#### 4.4 錯誤類型分類與統計

**適用場景**：需要系統性理解模型的錯誤模式

**設計方法**：

1. 收集模型的所有錯誤預測
2. 定義錯誤類型分類體系（可基於語言學、領域知識等）
3. 對錯誤進行人工分類
4. 統計各類型錯誤的比例

**分析要點**：

- 最常見的錯誤類型是什麼？
- 你的方法是否減少了特定類型的錯誤？
- 是否有新型態的錯誤被引入？

**呈現方式**：

```
表：錯誤類型分析

錯誤類型        | Baseline | 我們的方法 | 改善
----------------|----------|-----------|------
語義理解錯誤     | 45%      | 28%       | -17%
邊界識別錯誤     | 25%      | 22%       | -3%
罕見類型錯誤     | 20%      | 30%       | +10%
其他            | 10%      | 20%       | +10%
```

## 設計 Ablation Study 的通用檢查清單

### 完整消融

- [ ] 所有新提出的組件都有對應的消融實驗
- [ ] 移除方式合理（有適當的替代方案）
- [ ] 結果表格清楚展示每個組件的貢獻
- [ ] 討論了組件間可能的交互效應

### 超參敏感度

- [ ] 選擇了最關鍵的 2-4 個超參數
- [ ] 取值範圍合理且覆蓋足夠
- [ ] 繪製了清晰的敏感度曲線圖
- [ ] 提供了超參數選擇的實用建議

### 跨資料集遷移

- [ ] 資料集選擇具有多樣性
- [ ] 在所有資料集上使用相同的評估協定
- [ ] 分析了不同資料集上效能差異的原因
- [ ] 討論了方法的適用範圍與限制

### 定性分析

- [ ] 案例選擇具有代表性（非 cherry-picking）
- [ ] 同時展示成功與失敗案例
- [ ] 視覺化圖表清晰易讀
- [ ] 定性分析與定量結果相呼應
